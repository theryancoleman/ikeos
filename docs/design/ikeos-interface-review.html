Insight · Design Review · 11 Jun 2026

ʻIkeOS interface review — bolder, on-brand, one system.

The interface inside obsidian-capture works. It loads, it captures, the three-column workspace lands. But it doesn't look or feel like ʻIkeOS yet — it looks like a competent generic dark tool wearing the wrong purple. Below: an audit of where the gap sits, then a redrawn workspace, dashboard, and nav showing how far one CSS file can take it.

Reviewed  theryancoleman/obsidian-capture@b0673d Against  ʻIkeOS Design System Surfaces  workspace, tasks, capture, base nav
01 · VERDICT

A working tool with no brand on it.

The three-column layout is good — keep it. The visual language is not — the canvas is the wrong shade of dark (blue-black, not deep plum), the purple is the wrong purple (and there are two of them, by accident), and there is no ʻokina, no eyebrows, no halo, no constellation. Every component is doing generic-dark-tool work where the brand asks for navigator work. The fix is a token swap plus eight small component patches, not a rebuild.

What this screen says about the brand

A blue-black canvas with a violet-leaning accent reads as generic dark IDE, not as a navigator's instrument. The wordmark says Obsidian Capture, not ʻIkeOS. The session badges are pill-borders in currentColor — cleanly executed, but doing decoration where the brand asks them to do meaning. Nothing here is wrong; almost nothing here is ʻIkeOS.

03 · MISALIGNMENTS

Twelve gaps between code and system.

Sorted roughly by visible impact. Most of these collapse into a single CSS swap; some are one-line copy edits; two are component patches.

# What Has today System asks for Why it matters
01 Dark canvas #1A1A2E / #1A1A1A #120A1F deep plum The canvas is the brand. Today it's two competing dark greys; the spec calls for a single night-sky plum that all dark surfaces tint from.
02 Brand accent #7C5CBF / #7C6AF7 #7C3AED vibrant purple Two purples in two stylesheets, neither of them brand purple. style.css reads dusty; workspace.css reads cold-violet. Memory is one specific hue.
03 Wordmark "Obsidian Capture" ʻIkeOS + constellation mark The product is ʻIkeOS. The lockup with the ʻokina + orbit mark should be the first thing you see; the current text wordmark gives the system no identity.
04 UI typeface system-ui, sans-serif Geist (UI) + Geist Mono (IDs/timestamps) System-ui defaults to San Francisco/Segoe and ships a different rhythm on every host. Geist is the brand's working font; Geist Mono is the brand's numeric font.
05 Display moments No serif anywhere DM Serif Display on titles + page H1 The brand's slow-reading character lives in the serif. The dashboard "Dashboard" and workspace title are the right places for it — never in chrome, but always on page H1.
06 Section labels UPPER, 0.05–0.08em tracking Eyebrow: 11px, 0.22em, semibold, lavender-tertiary The signature eyebrow is one of the brand's smallest and most consistent moves — column heads, panel labels, capture form labels all want it. Today they're almost-eyebrows.
07 Focus + selected state 2px solid border-color shift Celestial halo (--glow-purple-md) The brand's signature elevation is a halo, not a border. 0 0 24px rgba(124,58,237,0.35) on the selected session card is the right reading.
08 Type badges (note/idea/bug) arbitrary forest/navy/maroon Semantic palette: knowledge / memory / insight Today the badge colours are decorative. The system says colour = meaning: notes are knowledge (blue), ideas are memory (purple), bugs are insight (orange — "the thing demanding your attention").
09 Status badges (new/open/in-prog/done) Five hand-mixed plums + greys SemanticDot + label, in semantic colour Today five flat pills do the work. Replace with a SemanticDot (halo for active states, dim for closed) + a plain text label — less noise, more navigator.
10 Activity pills (thinking/working) Light purple + sky-blue, currentColor border Memory (thinking) + Knowledge (working) with pulse dot The colours are nearly right by accident. Lock them to the semantic tokens and pair with a 2.2s pulse SemanticDot so the system's ambient motion shows through.
11 Empty states "Select a session to view details" Constellation field + wayfinder copy The brand frames empty as the start of a journey, not a deficiency. "Choose a session, or chart a new one." over a ConstellationField reads as ʻIkeOS in three seconds.
12 Two parallel CSS files style.css + workspace.css, each redefining tokens One token import from ʻIkeOS Design System The two files have already drifted (different purples, different surface greys). Consolidate to @import "ikeos/styles.css" at the top of one app stylesheet. Components reference tokens only.

06 · TOKEN MIGRATION

The CSS swap, end to end.

Today, app/static/style.css and app/static/workspace.css each define their own token block at the top. Both blocks get deleted; both files start by importing the ʻIkeOS tokens and use semantic variables only. Concrete map:

App variable todayMap to (ʻIkeOS)What it's doing
--bg: #1A1A2E --bg-canvas #120A1F Page background. Use --bg-canvas on body; never set a hex.
--bg-pane: #000 --bg-inset #0B061A The terminal-output pane. --bg-inset is the brand's "sunk" surface; output green still reads against it.
--surface: #16213E --bg-surface #1B1130 Nav, cards, inputs. --bg-surface is plum-tinted, not navy.
--bg-panel: #141414 --bg-raised #261845 Mid column (detail) and right column (capture). One step lifted from canvas.
--accent (style): #7C5CBF --ike-vibrant-purple #7C3AED Brand purple. Use through --brand-primary — never reference the hex.
--accent (workspace): #7C6AF7 --brand-primary #7C3AED Same target. Both stylesheets converge on one alias.
--border: #2E2E2E --border-default Lavender-tinted at low alpha on dark — the brand's whisper border, never pure grey.
--text: #E0E0E0 --text-primary #F8F7FC Subtly warm white — reads as moonlight, not LCD grey.
--text-muted: #888 --text-tertiary #8B82A4 Dusk-lavender muted, sits inside the family. The current grey is hue-neutral.
--success: #4ADE80 --sem-discovery #2DD4BF "Discovery green" — active sessions, success toasts. Teal-warmer than the current lime.
--warn: #FB923C --sem-insight #FF7849 The brand's one accent. Triage counts, attention-needed states, hover for bugs.
--thinking: #A78BFA --ike-soft-lavender #B497FF Memory / agent-thinking. Already nearly correct — lock to the token.
--working: #38BDF8 --sem-knowledge #3B82F6 Working = retrieving knowledge. Slightly cooler indigo, on-system.
--danger: #F87171 --status-error #F87171 Already matches the system. Use the alias to inherit future changes.
system-ui, sans-serif var(--font-sans) → Geist One font-family declaration on body. Headings inherit.
monospace var(--font-mono) → Geist Mono Timestamps, IDs, tmux names, version string — everything currently `monospace`.
border-radius: 6px / 8px / 4px (mixed) var(--radius-md) 10px · --radius-lg 14px · --radius-pill Cards → lg, inputs/buttons → md, badges → pill. Drop the loose 6/4/8 mix.
Implementation shape

Drop both :root token blocks. Add a single import at the top of app/static/style.css:  @import url("../static/ikeos/styles.css"); (vendored from this repo's styles.css → tokens/ + base.css). workspace.css stops redeclaring --accent entirely. Every background: var(--bg) becomes var(--bg-canvas), and so on down the table.

08 · ROLLOUT

Three phases, each shippable on its own.

Phase 1 alone gets the app 80% to the brand — tokens land everywhere and the wordmark/nav rebrand. Phase 2 is the workspace component-level patches. Phase 3 adds the brand's ambient layers and the celestial loader.

PHASE 01

Tokens + identity

~1 day · visible everywhere
  • Vendor the ʻIkeOS tokens into app/static/ikeos/ (tokens/ + base.css + styles.css entry).
  • Delete the :root blocks from style.css and .workspace-page from workspace.css; import the ʻIkeOS entry once.
  • Rebrand the nav — "ʻIkeOS" wordmark in DM Serif, constellation mark in front, sunset-orange whisker under the active link.
  • Swap every variable per the migration table. No raw hexes anywhere in app CSS by the end.
  • Geist + Geist Mono + DM Serif Display via the @import already in the system; system-ui retired.
PHASE 02

Component patches

~1 day · workspace + tasks
  • Session card — halo on selected, SemanticDot on activity, plum-800 surface, lavender border.
  • Type badges — note → knowledge-blue, idea → memory-purple, bug → insight-orange. Use Badge tokens.
  • Status badges — collapse to SemanticDot + label (dim/halo states).
  • Tasks H1 — DM Serif Display, with eyebrow + mono subtitle. Section heads become real eyebrows.
  • Capture form — type as semantic chips, eyebrow labels, lavender focus halo on inputs, ⌘↵ shortcut help.
PHASE 03

Ambient + voice

~½ day · texture and polish
  • Constellation field behind workspace + tasks bodies at 30% opacity, seeded so it doesn't reshuffle on every render.
  • LoadingMark replaces the "Loading…" empty-state and any future spinner. The constellation as loader is one of the brand's strongest moves.
  • Voice patches — the eleven copy edits in section 07.
  • Reduced-motion — gate the field's drift on prefers-reduced-motion.
  • Favicon + app icon — use logo-ikeos-mark and logo-ikeos-appicon-dark from assets/.
09 · KEEP DOING

Things to not change.

A short list, because most reviews skip this and turn into rewrites.

✓ Three-column layout Sessions / Detail / Capture is the right architecture. The system has nothing better to suggest.
✓ Command-first panel Putting the cmd input above the live output, not below it, is correct and the recent design spec already got this right.
✓ URL-state for selected session Refresh returns you to your context — this is exactly the kind of quiet care the brand asks for. Don't lose it.
✓ Activity verbs (thinking, working) The choice to surface what the agent is doing in lowercase verbs — "thinking", "working" — is already on-brand. Lock the colours, keep the verbs.
✓ EasyMDE for body No reason to swap. Just override its toolbar/CodeMirror colours to the ʻIkeOS tokens (the current overrides already do most of this).
✓ Lucide is not yet in use, and that's fine The system specifies Lucide @ 1.5px stroke. The app currently uses no icons. Add them only where they earn their keep — nav, command-send button. Don't sprinkle.
